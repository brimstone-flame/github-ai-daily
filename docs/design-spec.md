# 设计规范

## UI 设计原则

- **简洁直观**：信息密度适中，一眼可扫完
- **层次分明**：标题 → 板块 → 条目，逐级递进
- **配色主题**：淡蓝色 (`#e3f2fd` 色系)

## 配色方案

| 用途 | 色值 | 说明 |
|------|------|------|
| 主题色 | `#42a5f5` | 蓝色主调，用于标题/强调 |
| 背景色 | `#e3f2fd` | 淡蓝背景（HTML 模板时使用） |
| 分隔色 | `#bbdefb` | 浅蓝分隔线 |
| 文字色 | `#263238` | 深蓝灰，保证可读性 |
| 链接色 | `#1565c0` | 深蓝，可点击链接 |

## PushPlus 消息格式

- **当前阶段**：Markdown 格式（PushPlus 原生支持）
- **后续升级**：HTML 模板（可使用内联 CSS 实现淡蓝背景）

## Markdown 格式规则

1. 板块标题使用 `##` 二级标题 + emoji 前缀
2. 条目使用数字序号 `1.` 或无序列表 `•`
3. 链接使用 Markdown 语法 `[文本](URL)`
4. 分隔使用三个及以上 `━` 字符或 `---`
5. emoji 用作视觉标记，不使用 CSS
